## 2026-05-10 - [Inline Confirmations and CSS Transitions]\n**Learning:** When adding inline confirmations, using hardcoded inline styles (e.g. `style={{ background: ... }}`) will override base CSS transitions (`transition: all 0.2s`), causing abrupt, janky visual state changes. Also, `aria-expanded` is semantically incorrect for simple confirmation buttons.\n**Action:** Always toggle CSS module classes (e.g. `styles.clearBtnConfirm`) instead of using inline styles to preserve smooth transitions. Always ensure accessibility via dynamic `aria-label` without altering the button's role.
