## 2024-05-18 - [Destructive Action Confirmations & Package Manager Clashes]
**Learning:** Using `npm init -y` to set up temporary test environments (like Playwright) within a strictly `pnpm`-managed Next.js project will silently inject `"type": "commonjs"` into `package.json`, instantly breaking the Next.js ESM build environment. Additionally, inline confirmations for destructive actions (like "Clear all") provide superior UX to blocking dialogs but require robust cleanup of timeout references via `useEffect` to prevent memory leaks during rapid navigation.
**Action:** Never fall back to `npm` commands, even for temporary scaffolding, to avoid unintended configuration drift. Always implement `clearTimeout` in cleanup functions when using temporary UI states to maintain performance and avoid React state errors on unmounted components.
