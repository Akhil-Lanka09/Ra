## 2024-10-27 - Inline Confirmation State Updates\n**Learning:** Avoid changing the `role` attribute dynamically to `alert` on a button mid-interaction, as it can confuse screen readers and is considered an accessibility anti-pattern. Relying on explicitly toggling CSS modules (rather than inline styles that fight base transitions) ensures smooth animations without visual jarring.\n**Action:** Update the button text to clearly indicate confirmation, apply `aria-label`s to explicitly describe the expected action upon confirmation, and use scoped CSS class modifiers to provide visual feedback for destructive actions.
